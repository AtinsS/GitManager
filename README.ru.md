# 🚀 Git Менеджер Репозиториев от AtinsS

**Версия:** 1.0  
**Платформа:** Windows  
**Язык:** Batch Script

Удобный консольный менеджер для тех кто устал вводить команды Git вручную.
Позволяет легко управлять несколькими репозиториями через простое меню. ТОЛЬКО ДЛЯ WINDOWS!

## 📋 Возможности

### Основные функции:
- 📁 Управление списком репозиториев (добавление, удаление, редактирование)
- 🔄 Массовое обновление всех репозиториев
- 🌐 Клонирование новых репозиториев
- 📊 Просмотр статуса Git
- ⬇️ Git pull (обновление)
- ⬆️ Git push (отправка)
- 📝 Commit с комментарием
- ⚡ Быстрый commit + push с авто-комментарием
- 📜 Просмотр истории (git log)
- 🌿 Создание и переключение веток
- 🤖 Автоматические коммиты по расписанию

### Дополнительно:
- 🎨 Цветное оформление интерфейса
- 💾 Сохранение списка репозиториев в конфиг файл
- 🔍 Проверка наличия Git
- 🛡️ Обработка ошибок и предупреждений
- 👅 Двуязычная система определяет язык автоматически при запуске

## 📦 Требования

- Windows 10/11 (с поддержкой ANSI цветов)
- Git for Windows ([скачать](https://git-scm.com/download/win))
- Windows Terminal (рекомендуется) или любой современный терминал

## 🔧 Установка и использование

1. **Нужен Git**
2.  Скачайте и запустите
```batch
GIT-MANAGER.BAT
```
### Главное меню:
```
╔══════════════════════════════════════════════════════════╗
║              GIT МЕНЕДЖЕР РЕПОЗИТОРИЕВ                   ║
╚══════════════════════════════════════════════════════════╝

Сохраненные репозитории:
------------------------
1. my-project - C:\Projects\my-project (тут будет пусто пока не не будет создан репо)
2. work-repo - D:\Work\repository

Действия:
========
Введите номер репозитория (1-2)
C. Клонировать новый репозиторий
A. Добавить существующий локальный репозиторий
U. Обновить все репозитории
D. Удалить репозиторий из списка
S. Настройки
X. Выход
```

### Меню репозитория:
```
════════════════════════════════════════════════════════════
      Репозиторий: my-project
      Путь: C:\Projects\my-project
════════════════════════════════════════════════════════════

1. Git status (проверить состояние)
2. Git pull (обновить)
3. Git add + commit (с комментарием)
4. Git push (отправить)
5. Быстрый commit + push (авто-комментарий)
6. Просмотр истории (git log)
7. Создать ветку
8. Переключить ветку
9. Авто-коммиты (каждые N минут)
0. Вернуться в главное меню
```

## 📁 Структура файлов

### `git-manager.bat`
Основной файл менеджера с интерфейсом и логикой

### `git_repos.cfg`
Конфигурационный файл со списком репозиториев в формате:
```
имя_репозитория;полный_путь_к_репозиторию
```

### Папка `git-scripts/`
Содержит отдельные скрипты для каждой Git операции:

| Файл | Описание |
|------|----------|
| `01-git-status.bat` | Показывает состояние репозитория |
| `02-git-pull.bat` | Обновляет репозиторий |
| `03-git-commit.bat` | Делает commit с пользовательским комментарием |
| `04-git-push.bat` | Отправляет изменения |
| `05-git-quick-push.bat` | Быстрый commit с авто-комментарием + push |
| `06-git-log.bat` | Показывает историю коммитов |
| `07-git-create-branch.bat` | Создает новую ветку |
| `08-git-switch-branch.bat` | Переключает ветки |
| `09-git-auto-commit.bat` | Автоматические коммиты по расписанию |

## 💡 Примеры использования

### Добавление нового репозитория:
1. Выберите `C` для клонирования или `A` для добавления существующего
2. Введите имя репозитория
3. Укажите URL (для клонирования) или путь (для существующего)
4. Репозиторий появится в списке

### Быстрый commit + push:
1. Выберите репозиторий из списка
2. Нажмите `5`
3. Изменения будут автоматически закоммичены с комментарием "Quick update YYYY-MM-DD HH:MM" и отправлены

### Автоматические коммиты:
1. Выберите репозиторий
2. Нажмите `9`
3. Укажите интервал в минутах
4. Скрипт будет автоматически делать коммиты каждые N минут

## ⚠️ Возможные проблемы и решения

### Не работают цвета:
- Используйте Windows Terminal вместо стандартной консоли
- Или включите поддержку ANSI в реестре

### Git не найден:
- Установите Git for Windows
- Добавьте Git в переменную PATH

### Ошибка "Папка не найдена":
- Проверьте путь в конфиге
- Используйте опцию "Указать новый путь"
